# Best-Practices Docs Review — 2026-06-11

Verification of all four `BEST-PRACTICES.md` variants against official Next.js 16 (nextjs.org/docs, v16.2.9) and NestJS 11 (docs.nestjs.com) documentation, plus TanStack Query, Ant Design v6, Sentry, PostHog, Auth.js, Mongoose, and BullMQ sources. Each doc was reviewed in full by an independent reviewer with live doc lookups. Severity scale: **CRITICAL** (factually wrong / broken code / silent security failure), **OUTDATED**, **IMPROVEMENT**, **MINOR**.

**Overall verdict:** the Next 16 core guidance (caching model, `proxy.ts`, async request APIs, `use cache`/`cacheLife`/`cacheTag`, `revalidateTag`/`updateTag` semantics) and the Nest 11 core guidance (cache-manager/Keyv, throttler v6 syntax, `ConsoleLogger json`, Express 5 wildcards, argon2id params, auth Option A) are accurate and current. The docs are **not ready to adopt as-is**: 22 critical findings total, most of which are copy-pasteable broken code or features that silently do nothing.

## Cross-cutting issues (appear in multiple docs)

### Both Next.js variants

1. **CRITICAL — Module-level `QueryClient` singleton** (`lib/queryClient.ts`, ~675–688 in both). A file-root `QueryClient` is shared across server requests during SSR → cross-user cache/data leakage. TanStack's Advanced SSR guide mandates `makeQueryClient()` + `getQueryClient()` with `isServer` (new client per server request, lazy singleton in browser). The docs' own HydrationBoundary examples already create per-request clients, contradicting this section. https://tanstack.com/query/v5/docs/framework/react/guides/advanced-ssr

2. **CRITICAL — `PostHogPageView` calls `useSearchParams()` with no Suspense boundary**, mounted bare in `app/layout.tsx` (~1388/1408). Production build fails on any static route ("Missing Suspense boundary with useSearchParams"); passes in dev, breaks `next build`. Fix: `<Suspense fallback={null}><PostHogPageView /></Suspense>`. https://nextjs.org/docs/app/api-reference/functions/use-search-params

3. **CRITICAL — `useUser` hook fails `npm run typecheck`** (~724–737). Key factory declares `detail: (id: string)` but the hook passes `string | undefined`, then uses `fetchUser(id!)` violating the docs' own no-assertions rule. Fix: `queryFn: id ? () => fetchUser(id) : skipToken` (TanStack v5) and widen/guard the factory.

4. **CRITICAL (without-antd; applies to both) — Auth enforced only in `proxy.ts`**; Server Actions validate input but never check the session. Official guidance: proxy checks are *optimistic* only; every Server Action / data function must re-verify (`const session = await auth()` via a shared `requireUser()`); cf. CVE-2025-29927. https://nextjs.org/docs/app/guides/data-security

5. **OUTDATED — Sentry file convention**: docs say wizard creates `sentry.*.config.ts`; since Next 15.3 / SDK v9 it's `instrumentation-client.ts` + `instrumentation.ts` with `onRequestError` → `Sentry.captureRequestError`. Also `global-error.tsx` template drops the `error` prop, so nothing is reported from it.

6. **IMPROVEMENT — Per-call `useQuery<TData, TError>` generics**: TanStack recommends inference + global `Register { defaultError: ApiError }` module augmentation. Related: axios interceptor rejects a plain object, not an `Error` subclass — loses stacks, breaks `instanceof` and Sentry grouping. Use `class ApiError extends Error`.

7. **IMPROVEMENT — CSP misses PostHog assets host**: `posthog-js` lazy-loads replay/surveys bundles from `https://us-assets.i.posthog.com`, absent from `script-src` → features silently fail. Also add `frame-ancestors`, `object-src 'none'`, `base-uri 'self'`, `form-action 'self'`.

8. **IMPROVEMENT — Hardcoded `token.expiresAt = Date.now() + 14 * 60 * 1000`**: derive TTL from the login/refresh response; note the refresh-race with rotating tokens (Auth.js guide flags it).

9. **IMPROVEMENT — Server Actions section ignores React 19 primitives**: `useActionState` (pending/error consumption of returned results), `useOptimistic` ("don't need optimistic UI" claim is wrong — actions pair fine with it).

10. **IMPROVEMENT — Client/server boundary by prose only**: mandate `import "client-only"` in `lib/api/client.ts` and `"server-only"` in server data modules to make wrong imports build errors.

11. **MINOR (both)** — SSE handler `(payload as any)` violates own no-`any` rule; `revalidateTag` single-arg parenthetical slightly misstates equivalence vs `updateTag`; shadcn `DataTable` is a recipe, not a shipped component (without-antd).

### Both NestJS variants

12. **CRITICAL — Sentry setup captures nothing.** `SentryGlobalFilter` registered "before" the catch-all `HttpExceptionFilter`, claiming it "delegates to the next filter." Nest filters do not chain — exactly one matching filter runs; the catch-all `@Catch()` wins and Sentry never executes. Sentry's NestJS guide: with a global catch-all filter, do NOT use `SentryGlobalFilter`; decorate the filter's `catch()` with `@SentryExceptionCaptured()` instead. https://docs.sentry.io/platforms/javascript/guides/nestjs/

13. **CRITICAL — SSE service injects the same Redis client twice** (`@InjectRedis()` ×2 → same singleton). After `sub.subscribe()`, the shared connection enters subscriber mode and rejects all normal commands — breaks `publish`, SSE tickets (`GETDEL`), idempotency (`SET NX`), Redlock. Fix: `this.sub = this.redis.duplicate()` in `onModuleInit` (quit in `onModuleDestroy`) or a second named connection. Also: the module providing `@InjectRedis()` is never named/installed anywhere.

14. **CRITICAL — `SchedulerService` calls `this.authService.purgeExpiredRefreshTokens()` but never injects `authService`** — `TypeError` at runtime / fails `strict` compile.

15. **CRITICAL-adjacent — Global `JwtAuthGuard` lacks the `Reflector`/`IS_PUBLIC_KEY` check** that `@Public()` depends on; as written, login/health/SSE routes 401 and login is unreachable. `@Public()`/`@CurrentUser()` decorators are referenced but never defined. Guard naming also inconsistent (`AuthGuard` vs `JwtAuthGuard`, collides with Passport's export).

16. **OUTDATED/GAP — Express 5 query parser change omitted**: Nest 11 ships Express v5, default parser is now "simple" — nested/array query strings (`?filter[a]=1`) no longer parse without `app.set('query parser', 'extended')`. Directly affects the docs' pagination/sort query DTOs. https://docs.nestjs.com/migration-guide

17. **IMPROVEMENT — Unvalidated `sort` field** passed straight into the ORM/ODM `order`/`sort` — any entity property sortable (incl. `passwordHash`; timing/ordering side channel, 500s on unknown fields, unindexed-field DoS). Whitelist via `@IsIn([...])`.

18. **IMPROVEMENT — Internal contradiction on global registration**: Modules section mandates `APP_PIPE`/`APP_INTERCEPTOR` for anything with dependencies, then validation section uses `useGlobalPipes()` and `useGlobalInterceptors(new ClassSerializerInterceptor(app.get(Reflector)))`. Show both as `APP_*` providers.

19. **IMPROVEMENT — DIP: `FilesService`/`AnalyticsService` construct SDK clients (`new S3Client`, `new PostHog`) in constructors.** Provide via factory provider (`{ provide: R2_CLIENT, useFactory: createR2Client, inject: [ConfigService] }`) and inject — consistent with the docs' own token guidance, makes unit tests trivial.

20. **IMPROVEMENT — `transform: true` claim overstated**: auto-coercion applies to `@Param()`/`@Query()` handler primitives; DTO members need `@Type(() => Number)` or `enableImplicitConversion`.

21. **MINOR — Redlock v5** is a years-long beta with minimal maintenance; prefer BullMQ repeatable jobs (already distributed-safe) or, in the PG variant, `pg_try_advisory_lock`.

22. **GAPS (both)**: refresh-token browser transport (httpOnly/Secure/SameSite cookie + CSRF) never specified; Swagger CLI plugin (`nest-cli.json`) unmentioned; Nest 11's `ParseDatePipe` and `IntrinsicException` absent; body size limits (`json({ limit })`) absent; request-lifecycle ordering (middleware → guards → interceptors → pipes → handler → interceptors → filters) never stated though several patterns depend on it.

## frontend/next/with-antd — variant-specific

- **CRITICAL — shadcn/Radix/Sonner/React-Hook-Form contamination.** Project tree says "primitives (shadcn)"; all `loading/error/not-found/global-error` templates import `@/components/ui/*`; Turnstile login form uses RHF + shadcn `<Form>` (while the antd Forms section forbids manual state and mandates antd `Form`); SSE handler uses Sonner `notify` (while Error Handling mandates `App.useApp()` notification "instead of Sonner"); Accessibility section recommends Radix/shadcn primitives. Two mutually exclusive form stacks and two toast systems in one standard. Rewrite with antd equivalents (`Skeleton active`, `Result status="404"/"500"`, antd `Form`, `App.useApp().notification`).
- **CRITICAL — Missing `@ant-design/nextjs-registry`.** Root layout shows only `<ConfigProvider>`. Ant Design's App Router guide requires `<AntdRegistry>` for SSR style extraction (still required in v6/CSS-variables mode) — without it, FOUC/unstyled SSR. https://ant.design/docs/react/use-with-next
- **CRITICAL — Lead App Router example imports the client-only axios layer into a Server Component** (`getUsers` from `@/lib/api/users` — which actually exports `fetchUsers`): silently unauthenticated prefetch (no cookie context for `getSession()`), and violates the doc's own Axios-section rule. Use a server-side fetcher with `await auth()`.
- **CRITICAL — Nonexistent antd API**: "Use `placement` prop on columns (not `position` — renamed in v6)" — Table columns never had `position`/`placement`; no such rename in the v6 migration guide. Delete (or document `pagination={{ position: [...] }}` if that was the intent). *(Verify once manually against https://ant.design/components/table — finding rests partly on absence of evidence.)*
- **OUTDATED — "Removed in v6" overstates**: `Button.Group`/`Input.Group`/`BackTop`/`headStyle`/`bodyStyle`/`dropdownRender`/`onDropdownVisibleChange` are **deprecated** in v6 (removal in v7); v6 is "fully compatible with v5". Recommended replacements are all correct.
- **OUTDATED — Modal/Drawer mask blur**: v6 reverted blur-by-default for GPU perf (ant-design PR #56781); it's now opt-in. Drop the `backdropFilter: "none"` workaround.
- **IMPROVEMENT — Turnstile invisible test key**: invisible always-pass dummy key is `1x00000000000000000000BB` (the doc's `…AA` is the visible widget).
- **MINOR**: `ConfigProvider theme` from a server layout breaks the moment `algorithm: theme.darkAlgorithm` (a function) is added — recommend a `"use client"` providers file; project tree omits `proxy.ts`/`auth.ts`; prefer root `TableColumnsType` export over `antd/es/table` deep import; antd + jsdom tests need a `window.matchMedia` mock.

## frontend/next/without-antd — variant-specific

- **IMPROVEMENT — "loading.tsx always, every segment" contradicts the `cacheComponents` model the doc recommends**: route-level `loading.tsx` replaces the whole prerendered shell and re-triggers per navigation; prefer granular `<Suspense>` islands, keep `loading.tsx` as a safety net.
- **IMPROVEMENT — Internal contradiction**: root layout imports `Toaster` from `sonner` two paragraphs before "Never import `sonner` outside `lib/notify.ts`". Re-export `Toaster` from `lib/notify.ts`.
- **MINOR**: `cacheLife` comment omits `default`/custom profiles; `revalidateTag(tag, { expire: 0 })` webhook escape hatch worth adding; `gen:api` script uses `$VAR` (breaks on Windows); "per tick" dedupe wording; pin exact `next-auth@beta` version.

## backend/nest/with-pg — variant-specific

- **CRITICAL — `ThrottlerGuard` never bound.** Only `ThrottlerModule.forRoot([...])` shown; without `{ provide: APP_GUARD, useClass: ThrottlerGuard }` every rate limit in the doc (incl. auth brute-force limits) is silently inert. Note APP_GUARD registration order vs the JWT guard. https://docs.nestjs.com/security/rate-limiting
- **CRITICAL — `.env.test` never loaded**: `setupFiles: ["dotenv/config"]` + "dotenv loads .env.test when NODE_ENV=test" is false (that's dotenv-flow/Next behavior) — e2e tests hit the **development** DB, violating the doc's own rule. Use a setup file with `dotenv.config({ path: ".env.test" })` or `ConfigModule.forRoot({ envFilePath: ".env.test" })`.
- **CRITICAL — TypeORM CLI snippets are mutually broken**: `-d dist/data-source.js` with `entities: ["src/**/*.entity.ts"]` globs → CLI loads zero entities (or regenerates the whole schema); migrations dir differs across CLI command, runtime config, and CLAUDE.md (three paths). Show one coherent end-to-end setup (ts-node CLI + src globs, or build-first + `__dirname`-relative dist globs).
- **OUTDATED — `validationSchema` conflated with class-validator**: `validationSchema` is Joi-only; class-validator uses the separate `validate` function option. https://docs.nestjs.com/techniques/configuration
- **IMPROVEMENT — SOLID/SRP self-contradiction: `bcrypt.hash(dto.password, 12)` inline in three `UsersService` snippets** while the auth section mandates hashing behind the auth service (ADR-005). Extract a `PasswordHasher` port (interface + token) — also makes the promised bcrypt→argon2id swap a one-line change.
- **IMPROVEMENT**: `autoLoadEntities: true` over entity globs; `version: VERSION_NEUTRAL` on health controller (else probes hit `/v1/health`); e2e `overrideProvider(ConfigService)` partial mock → let real ConfigModule load `.env.test`; Testcontainers promised in CLAUDE.md but absent from the doc; PG `enum` column type for `role` instead of varchar.
- **MINOR**: `findAndCount` is two queries, not "a single round-trip"; `@Index(["email"])` redundant with `unique: true`.
- **GAPS**: connection-pool sizing (replicas × pool ≤ `max_connections`, PgBouncer), optimistic/pessimistic locking & isolation levels, N+1/relations strategy, seeding.

## backend/nest/without-pg (MongoDB) — variant-specific

- **CRITICAL — `passwordHash` leaks in the reference implementation.** Controllers section says "return the document, `ClassSerializerInterceptor` strips `@Exclude()`" — but the doc's own later caveat (correct) says `@Exclude()` is NOT honored on Mongoose `HydratedDocument`s, and the `@Schema()` example has no `toJSON` transform. The shown e2e assertion (`expect(body.passwordHash).toBeUndefined()`) would fail against the shown code. Make the safe pattern primary: schema-level `toJSON` transform + `.select("-passwordHash")`, or `plainToInstance(UserResponseDto, doc.toObject())`.
- **CRITICAL — "Use `interface` for DTOs" is wrong**: Nest DTOs must be classes (interfaces erase at transpile; ValidationPipe/swagger can't see them) — contradicts the doc's own `import type` warning and every DTO example in it. https://docs.nestjs.com/controllers#request-payloads
- **OUTDATED — custom `ParseObjectIdPipe`**: `@nestjs/mongoose` ships `ParseObjectIdPipe`/`IsObjectIdPipe` (also transforms to `Types.ObjectId`).
- **IMPROVEMENT**: manual `startTransaction/commit/abort` → `session.withTransaction()` (auto-retries `TransientTransactionError`); SQL-injection bullet is PG-variant leftover — replace with Mongo operator-injection guidance (`sanitizeFilter`, never pass raw request objects to `find()`); idempotency replay loses status/headers (store `{ status, body }`) and `tap()` doesn't await the Redis set; offset pagination → mention cursor/range for large collections.
- **MINOR**: Modules example uses `TypeOrmModule.forFeature([User])` (PG leftover); duplicate unique index on `email` (`@Prop({ unique })` + `schema.index()`); unit-test mock isn't constructable with `new this.userModel(...)`; `jwt.issuer`/`audience` read by guard but missing from config factory.
- **GAPS**: `autoIndex: false` in production; readiness vs liveness split for Terminus (liveness must be dependency-free).

## SOLID assessment (both backends)

**Strong:** DIP via Symbol tokens + ports where it matters (`MAIL_PROVIDER`), thin controllers (SRP), guards-own-authorization (SRP/OCP), the circular-dependency escalation ladder, the documented-and-reasoned exception for direct `Repository<T>`/Model injection, the "don't over-wrap" counterweight.

**Violations to fix:** self-constructed SDK clients in the designated wrapper services (DIP/testability, item 19); inline password hashing in `UsersService` (SRP + self-contradiction, with-pg); controllers typed to return persistence types (`Promise<User>` / `Promise<UserDocument>`) while Swagger documents response DTOs — wire contract and runtime shape drift independently (ISP/contract risk; root cause of the Mongo `passwordHash` leak); `refreshStore` used but its port never defined; `MailProcessor` switch (OCP, minor); `SchedulerService` mixes lock plumbing with job logic (SRP, minor).

No LSP issues found in any doc.

## Recommended fix order

1. **Silent security/observability failures**: ThrottlerGuard binding (with-pg), Sentry filter pattern (both Nest), proxy-only auth → Server Action checks (both Next), Mongo `passwordHash` serialization (without-pg), `.env.test` loading (with-pg).
2. **Broken-as-written code**: SSE Redis duplicate connection, scheduler injection, global guard `@Public()` check, TypeORM CLI setup, `useSearchParams` Suspense, `QueryClient` factory, `useUser` typing, with-antd Server-Component prefetch.
3. **Stack contamination (with-antd)**: shadcn/Sonner/RHF removal, `AntdRegistry`, antd v6 deprecated-vs-removed wording, mask-blur revert, fake `placement` prop.
4. **Consistency/SOLID**: APP_* registration style, PasswordHasher port, injected SDK clients, sort whitelists, DTO-vs-interface sentence, PG/Mongo leftovers.
5. **Gaps**: Express 5 query parser (both Nest), Turbopack/React Compiler/`useActionState` notes (both Next), refresh-token transport, remaining items above.
